# TinyMQ

轻量级分布式消息队列，基于 Go 语言实现，采用单 Broker 架构、LevelDB 索引存储、主从复制与 Consumer Group 负载均衡。

## 架构图（文字版）

```
+------------------+       +------------------+
|    Producer      |       |    Consumer      |
|   (生产者API)     |       |   (消费者API)     |
+--------+---------+       +--------+---------+
         |                          |
         v                          v
+--------+--------------------------------------+
|                     API 层                     |
|  (Producer API / Consumer API / HTTP+TCP)     |
+--------+--------------------------------------+
         |
         v
+--------+--------------------------------------+
|                   Broker 层                    |
|  +-----------------------------------------+  |
|  |  Topic A  (Partition 0 / Partition 1)   |  |
|  +-----------------------------------------+  |
|  +-----------------------------------------+  |
|  |  Topic B  (Partition 0 / Partition 1)   |  |
|  +-----------------------------------------+  |
|  +-----------------------------------------+  |
|  |  Consumer Group (负载均衡 / ACK / 重试)  |  |
|  +-----------------------------------------+  |
+--------+--------------------------------------+
         |
         v
+--------+--------------------------------------+
|                  Storage 层                    |
|  +----------------+  +---------------------+  |
|  |  Index (LevelDB)|  |  Log File (消息体)  |  |
|  |  offset -> pos  |  |  顺序追加写入        |  |
|  +----------------+  +---------------------+  |
+--------+--------------------------------------+
         |
         v
+--------+--------------------------------------+
|               Replication 层                   |
|  +----------------+  +---------------------+  |
|  |    Master      |->|      Slave          |  |
|  |  (主节点写入)   |  |  (从节点同步/故障切换)|  |
|  +----------------+  +---------------------+  |
+------------------------------------------------+
```

## 核心模块

- **broker/**: 消息路由、Topic/Queue 管理、Consumer Group 与 ACK 超时重试
- **storage/**: LevelDB 索引 + 顺序日志文件，支持随机读与批量顺序读
- **replication/**: 主从数据同步、故障检测与 Leader 自动切换逻辑
- **api/**: Producer/Consumer 接口，基于 HTTP 暴露
- **demo/**: 生产者和消费者示例程序

## 快速开始

```bash
cd demo/producer && go run main.go
cd demo/consumer && go run main.go
```
